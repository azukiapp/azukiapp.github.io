Azkfile.js

Azkfile.js is the file format for configuration recipes that inform AZK what to do. An Azkfile.js is a short and simple file written in its own easy-to-understand DSL.

Once the desired Azkfile.js is ready, it’s as simple as “just push play”.

  Dependent systems
Mounts folders to assigned paths
Configure Database
  systems({
    'node-example': {
      depends: ["database"],
      image: "dockerfile/nodejs",
      provision: [
        "npm install",
      ],
      workdir: "/azk/" + manifest.dir,
      command: "node index.js",
      mount_folders: {
        '.': "/azk/::system::",
      },
      scalable: { default: 3 },
      http: {
        hostname: url("::system::", default_domain),
      },
      envs: {
        NODE_ENV: "dev",
      }
    },

    database: {
      image: "azukiapp/redis",
      ports: {
        data: "6379/tcp"
      }
    }
  });

  defaultSystem('node-example');
  addBin("grunt", ["--system", "node-example", "-c", "grunt", "-t"]);

AZK Features

What’s so great about AZK?

Open Source

AZK is open source software (Apache 2.0) and will always be.

Flexibility

AZK is agnostic. Keep using the exact same development tools we already used before.

Reuse

Reuse entire recipes ou just parts of them. Add new components or create new recipes from scratch. Customize everything according to your requisites. It’s as easy as editing a text file.

Automation

Install and configure development environments with a few commands. It will only take minutes instead of hours or days.

AZK does its magic by executing very short and simple recipe files (Azkfile.js) that describe the environments to be installed and configured.

Performance

AZK uses containers instead of virtual machines. Containers are like virtual machines, only with better performance and lower consumption of physical resources. It’s fast and your machine will barely feel it.

AZK is built with Docker (http://docker.io), an open source engine for using containers.

Best Practices

AZK characteristics make the adoption of software development best practices a natural and spontaneous process.

Sharing recipes assures total parity among development environments of every programmer involved in the same project. Lower the probability of bugs coming up during deployment.

Aren’t you sure if all programmers in your team are using the development environment latest version? Just give them the latest configuration recipe.

With AZK, you can kill and start entire development environments in seconds (literally).

Use Cases

AZK is for everyone in a software development project. Find out what it can do for you:

  • Software Developers
  • Sys Admin/Dev Ops
  • Designers, managers etc.

Software Developers

All you need to install and configure your development environment in a few moments is the corresponding Azkfile.js. You can write it yourself from scratch, edit an existing one or, if you already have any code written, tell AZK to create a new one for that code (and you can even further edit it from this point on if you want to).

You are a coder. All you should have to do is code. ;)

Sys Admin/Dev Ops

Use AZK not only to automate the installation and configuration workflow of development environments for your team. With AZK, you can also maintain uniformity across those environments with no effort. Also, the way AZK builds environments makes it super easy to keep track of the evolution of their requisites.

You’ll be your team’s hero. 8)

Designers, managers etc.

(Soon). With the graphical user interface module, every project stakeholder that has to run code in development for any reason will be able to do so even if they can’t program or install and configure development environments.

It doesn’t matter what kind of tests or checks you have to perform (UX, design, QA etc.), with the correct Azkfile.js you can install everything (including the project code itself) with literally just a few clicks.

You’ll always be empowered. \o/
