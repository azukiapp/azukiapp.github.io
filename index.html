AZK Features

What’s so great about AZK?

Open Source

AZK is open source software (Apache 2.0) and will always be.

Flexibility

AZK is agnostic and has a very soft learning curve. Keep using the exact same development tools you already use.

Automation

Install and configure development environments with just a few commands. It will only take minutes instead of hours or days.

AZK does its magic by executing very short and simple recipe files (Azkfile.js) that describe the environments to be installed and configured.

Reuse

Reuse an entire Azkfile.js or just parts of it. Add new components or create new ones from scratch. Customize everything according to your requisites.

It’s as easy as editing a text file.

 

Performance

AZK uses containers instead of virtual machines. Containers are like virtual machines, only with better performance and lower consumption of physical resources. It’s fast and your machine will barely feel it.

AZK is built with Docker, an open source engine for managing containers.

Best Practices

Sharing an Azkfile.js assures total parity among development environments in different programmers machines and reduces the chances of bugs during deployment.

Aren’t you sure if all the programmers in your team are using the updated version of the development environment? Just give them the latest Azkfile.js. With AZK, you can kill and restart entire development environments in seconds (literally).

Azkfile.js

Azkfile.js is the file format for configuration recipes that inform AZK what to do. An Azkfile.js is a short and simple file written in its own easy-to-understand DSL.

Add elements that compose your system and describe depedencies among them.

Specify requisites for running your system.

Tell how and where we must run your system.

Give your system an URL and set the inital number of its instances.

Add configuration parameters.

Define persistent mounting points.

Map, name and tipify (udp/tcp) the ports you want to use.

  // AZK 0.4.0

  systems({
    'web': {
      depends: ["database"],
      image: "dockerfile/nodejs",
      provision: [
        "npm install",
      ],
      command: "node index.js",
      workdir: "/azk/<%= manifest.dir %>",
      mount_folders: {
        '.': "/azk/<%= manifest.dir %>",
      },
      scalable: { default: 3 },
      http: {
        hostname: "<%= system.name %>.<%= azk.default_domain %>",
      },
      envs: {
        NODE_ENV: "dev",
      }
    },
    database: {
      image: "azukiapp/redis",
      persistent_folders: ["/data"],
      ports: {
        data: "6379/tcp"
      }
    }
  });

  // AZK 0.4.0

  // Add elements that compose your system and describe depedencies among them.
  systems({
    'web': {
      depends: ["database"],
      // Specify requisites for running your system.
      image: "dockerfile/nodejs",
      provision: [
        "npm install",
      ],
      // Tell how and where we must run your system.
      command: "node index.js",
      workdir: "/azk/<%= manifest.dir %>",
      mount_folders: {
        '.': "/azk/<%= manifest.dir %>",
      },
      // Give your system an URL and set the inital number of its instances.
      scalable: { default: 3 },
      http: {
        hostname: "<%= system.name %>.<%= azk.default_domain %>",
      },
      // Add configuration parameters.
      envs: {
        NODE_ENV: "dev",
      }
    },
    database: {
      image: "azukiapp/redis",
      // Define persistent mounting points.
      persistent_folders: ["/data"],
      // Map, name and tipify (udp/tcp) the ports you want to use.
      ports: {
        data: "6379/tcp"
      }
    }
  });

Use Cases

AZK is for everyone in a software development project. Find out what it can do for you:

Software Developers

Get the right Azkfile.js and AZK does the rest. You’ll have your development environment installed and configured for you in a few moments. You can create an Azkfile.js from scratch, edit an existing one or even tell AZK to suggest you one based on your requirements.

You are a coder. All you should have to do is to code. ;)

Sys Admin/Dev Ops

Use AZK to automate the installation and configuration workflow of development environments for your team. You’ll also be able to maintain uniformity across those environments with no effort. Finally, the way AZK builds environments makes it super easy to keep track of their evolving versions.

You’ll be your dev team’s hero. 8)

Designers, UX, QA, product managers etc.

(Soon). With the graphical user interface module, every project stakeholder that has to run code in development for any reason will be able to do so, even if they can’t program or install and configure development environments.

It doesn’t matter what kind of tests or checks you have to perform (UX, design, QA, product management etc.). With the project’s Azkfile.js you can install everything (including the project code itself) with literally just a few clicks.

You’ll always be empowered. \o/
